Functions

  • Do not Repeat Yourself (DRY) principle- If the code gets repeat capture it into a function.
  • Modular code- Create a functionality and use it anywhere
  • Changes made only at one place.
  • first class members- Function can be stored in any variable.
  • In javascript every function return something.

Syntax

Defination-

function fname(){
}

Function invoke

fname();

Syntax

()=>{
}

Types

Function statement-

runs with function's name

function ramram(){
console.log("ram ram")
}

ramram();

Function expression-

store function statement in variable and now the variable has all the power of function

let fun = function game(){
console.log("ludo")
}

fun();

Anonymous Function-

function does not have name

let angel = function(){
console.log("I am angel priya")
}

angel();

Function with parameters-

parameters and arguments

function details(para1,para2,para3){
...kaam...
}

details(argument1,argument2,argument3);

function details(name,age,salary){
console.log(name,age,salary)
}

details("sparsh",24,5000);

Function with return keyword-

every function returns something and no code after return will run (like break keywork in loops). The return keyword needs to be catched in a variable.

function age(umar){
return umar;
console.log("nahi chalega")
}

let ans = age(24);

console.log(ans)

Types

Fat arrow function-

runs with variable's name

let fun=()=>{
console.log("hello sir")
}

fun();

Fat arrow with parameter-

parameters and arguments

let details=(name,age,salary)=>{
console.log(name,age,salary)
}

details("sparsh",24,5000)

Fat arrow with return-

every function returns something

let fun=(game)=>{
return game;
console.log("nahi chalega")
}

let ans=fun("ludo");
console.log(ans)

Fat arrow with implicit return-

no need to write return keyword

let number = (a,b)=>a+b;

let sum = number(10,20)
console.log(sum)

OR

console.log(number(10,20))

  • If their is only one parameter then don't use round () brackets.
  • If the function has only one statement then don't use curly {} brackets.
